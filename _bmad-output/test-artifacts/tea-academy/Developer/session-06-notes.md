## Key Concepts Covered

### 1. Test Review Workflow — 5 Dimensions of Quality
Overall Score = Average of all 5 dimensions (0-100 each)

**1. Determinism** — tests pass/fail consistently, no flakiness, proper async handling
**2. Isolation** — tests run independently, no shared state, fully parallelizable
**3. Assertions** — checking the right things meaningfully, fails for the correct reason
**4. Structure** — readable, well-named, clear Arrange-Act-Assert, minimal duplication
**5. Performance** — fast execution, efficient resources, parallelism exploited

### 2. Trace Workflow — Requirements Traceability
Steps:
1. Load Context — understand acceptance criteria
2. Discover Tests — find all relevant test files
3. Map Criteria — link each criterion to a test
4. Analyze Gaps — what criteria have no coverage?
5. Gate Decision — GREEN / YELLOW / RED

Release Gate:
| Gate | Condition | Decision |
|------|-----------|----------|
| 🟢 GREEN | All P0/P1 criteria covered | Ship it |
| 🟡 YELLOW | Some P1 gaps | Assess risk |
| 🔴 RED | Any P0 gaps | Do not ship |

Real example: password reset feature — verify "expired link returns 400" maps to a test before shipping.

### 3. Quality Metrics That Matter
**Track:**
- P0/P1 coverage % (the only coverage number that means something)
- Flakiness rate (target: 0%)
- Test execution time (feedback loop speed)
- Determinism score from Test Review (target: ≥90)

**Don't track (vanity metrics):**
- Total line coverage % — false confidence
- Number of tests — quantity ≠ quality
- Test file count — irrelevant
