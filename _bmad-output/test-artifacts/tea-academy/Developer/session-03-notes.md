## Key Concepts Covered

### 1. Fixture Architecture
- Fixtures solve the setup/teardown boilerplate problem
- Composable: build complex fixtures from simple base fixtures using `mergeTests()`
- Cleanup is automatic via the `use()` callback pattern
- Think of it like dependency injection — declare what you need, framework wires it up
- Knowledge Fragment: fixture-architecture.md, fixtures-composition.md

```typescript
const authFixtures = {
  authenticatedPage: async ({ page }, use) => {
    await page.goto('/login');
    await login(page);
    await use(page); // test runs here, cleanup runs after
  },
};
test.use(mergeTests(baseFixtures, authFixtures));
```

### 2. Network-First Patterns
- Set up route interception BEFORE triggering the action — never after
- Prevents race conditions (request may fire before mock is registered)
- Deterministic, fast, controllable
- Knowledge Fragment: network-first.md, intercept-network-call.md

```typescript
// ✅ Correct: intercept first, then act
await page.route('/api/users', (route) => route.fulfill({ json: mockUsers }));
await page.click('Load Users');

// ❌ Wrong: race condition
await page.click('Load Users');
await page.route('/api/users', ...);
```

### 3. Data Factories
- Factory functions replace hardcoded test data
- Only override what the test cares about — defaults handle the rest
- Self-documenting: `createUser({ role: 'admin' })` tells the story
- Knowledge Fragment: data-factories.md

```typescript
function createUser(overrides = {}) {
  return { id: faker.string.uuid(), email: faker.internet.email(), role: 'user', ...overrides };
}
const admin = createUser({ role: 'admin' });
```

### 4. Step-File Architecture
- Micro-file design: each step is self-contained
- Just-in-time loading: only current step in memory
- Sequential enforcement: no skipping
- State tracking: progress saved to YAML for resumability
- This very workflow uses step-file architecture

## Key Takeaways

- Fixtures = dependency injection for tests; compose them, don't repeat them
- Always intercept network BEFORE triggering the action (network-first)
- Data factories make tests self-documenting and decoupled from hardcoded data
- Step-file architecture enables disciplined, resumable, maintainable workflows
- These patterns directly address the "hard to maintain tests" pain point
