## Key Concepts Covered

### 1. ATDD Workflow (Red Phase First)
Steps:
1. Preflight — check prerequisites
2. Test Strategy — define what to test (from Test Design doc)
3. Generate Failing Tests — red phase, code doesn't exist yet
4. Implement Code — green phase, make tests pass

Why failing first:
- Proves the test actually tests something
- Prevents false positives
- Drives implementation — tests define the contract before the code

### 2. Automate Workflow (Coverage Expansion)
For code that already exists.

Steps:
1. Identify Targets — what features lack coverage?
2. Generate Tests — API and/or E2E
3. Review & Run — tests should pass immediately

| | ATDD | Automate |
|--|------|----------|
| When | Before code | After code |
| Test state | Start failing (red) | Start passing |
| Purpose | Drive implementation | Expand coverage |

### 3. Component TDD Red-Green Loop
1. Red — write failing test for next small behaviour
2. Green — minimal code to pass (don't over-engineer)
3. Refactor — clean up while keeping tests green
4. Repeat

```typescript
// RED
test('calculates total price', () => {
  expect(calculateTotal([10, 20])).toBe(30);
});

// GREEN (minimal)
function calculateTotal(prices: number[]): number {
  return prices.reduce((sum, p) => sum + p, 0);
}

// REFACTOR
function calculateTotal(prices: number[]): number {
  if (!prices.length) return 0;
  return prices.reduce((sum, p) => sum + p, 0);
}
```

### 4. API Testing Patterns
Pure API tests — no browser, fast, direct business logic access.

```typescript
test('POST /auth/forgot-password sends reset email', async ({ request }) => {
  const response = await request.post('/api/auth/forgot-password', {
    data: { email: 'user@example.com' }
  });
  expect(response.ok()).toBeTruthy();
  const body = await response.json();
  expect(body.message).toBe('Reset email sent');
});

test('returns 400 for unknown email', async ({ request }) => {
  const response = await request.post('/api/auth/forgot-password', {
    data: { email: 'nobody@nowhere.com' }
  });
  expect(response.status()).toBe(400);
});
```

## Key Takeaways

- ATDD: write failing tests first, let them drive your implementation
- Automate: systematic coverage expansion for existing code
- TDD loop: Red → Green → Refactor → Repeat (keep steps small)
- API tests skip the browser — faster feedback, direct logic access
- "Minimal code to pass" is the discipline — resist over-engineering in green phase
